# 云基础设施安全

用于审查云资源、部署配置、网络访问、密钥管理和权限策略。

## 身份与权限

- [ ] 使用最小权限原则
- [ ] 不使用长期 root/admin 凭证执行日常任务
- [ ] 服务账号只授予必要资源和动作
- [ ] CI/CD 凭证有明确作用域和过期策略
- [ ] 跨账号访问需要显式信任边界

## 密钥管理

- [ ] 密钥存放在云密钥管理服务或环境注入系统
- [ ] 不把 `.env`、证书、私钥提交到仓库
- [ ] 支持密钥轮换
- [ ] 日志和错误信息不输出密钥

## 网络边界

- [ ] 数据库、缓存、消息队列不直接暴露公网
- [ ] 安全组或防火墙规则限制来源和端口
- [ ] 管理端口只允许可信网络访问
- [ ] 对外服务启用 TLS
- [ ] 内部服务使用私有网络或服务网格策略

## 存储安全

- [ ] 对象存储默认私有
- [ ] 公共 bucket 有明确业务原因
- [ ] 开启服务端加密
- [ ] 生命周期策略符合数据保留要求
- [ ] 备份和快照权限受控

## 容器与镜像

- [ ] 镜像来自可信来源
- [ ] 基础镜像定期更新
- [ ] 容器不以 root 运行，除非必要
- [ ] 不在镜像层写入密钥
- [ ] 部署资源限制合理

## 监控与审计

- [ ] 开启访问日志和审计日志
- [ ] 关键权限变更有告警
- [ ] 异常登录、异常流量、失败认证可追踪
- [ ] 日志保留周期满足排障和合规需求

## 发布检查

- [ ] 生产和测试环境隔离
- [ ] 基础设施变更可审计
- [ ] 回滚路径明确
- [ ] 高危变更经过人工确认
- [ ] 已检查成本、配额和容量风险
