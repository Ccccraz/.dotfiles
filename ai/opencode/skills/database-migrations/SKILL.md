# 数据库迁移

用于规划、编写和审查数据库 schema 或数据迁移。

## 原则

1. 迁移必须可重复执行或有明确前置条件
2. 大表变更避免长时间锁表
3. 破坏性变更分阶段发布
4. 数据迁移要可观测、可回滚或可前滚
5. 应用代码和数据库变更要兼容滚动发布

## 安全发布模式

### 新增字段

1. 添加允许为空的字段
2. 部署写入新字段的应用
3. 回填历史数据
4. 增加约束或默认值
5. 删除旧字段读取逻辑

### 重命名字段

1. 添加新字段
2. 双写新旧字段
3. 回填新字段
4. 读路径切到新字段
5. 停止写旧字段
6. 删除旧字段

### 添加索引

PostgreSQL 大表优先使用：

```sql
CREATE INDEX CONCURRENTLY idx_name ON table_name (column_name);
```

注意：`CONCURRENTLY` 不能放在事务块中。

## 数据迁移检查

- [ ] 有批处理和进度记录
- [ ] 可从失败位置继续
- [ ] 有 dry-run 或预估影响
- [ ] 有回滚或补偿方案
- [ ] 迁移后有校验 SQL

## 回滚策略

- 优先设计可前滚修复
- 删除字段、删除表、不可逆数据变换必须额外确认
- 回滚脚本要和迁移脚本一起审查
- 对生产数据先备份或快照

## 审查清单

- [ ] 是否会锁大表
- [ ] 是否影响旧版本应用
- [ ] 是否需要后台回填
- [ ] 是否有唯一约束冲突风险
- [ ] 是否有索引膨胀或性能影响
- [ ] 是否记录迁移耗时和结果
