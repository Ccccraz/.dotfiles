# 迭代检索

用于在上下文有限时逐步获取信息，避免一次性读取过多文件或把无关内容带入任务。

## 原则

- 先宽后窄
- 每一轮检索都基于上一轮发现
- 只读取和当前决策相关的文件
- 记录关键事实和不确定点
- 不把猜测当事实

## 三轮模式

### 第一轮：定位

- 用文件名、目录和关键词找到候选区域
- 不急着读大文件
- 识别入口、测试、配置和相关模块

### 第二轮：理解

- 阅读最相关的 2-5 个文件
- 找到调用链、数据流和项目约定
- 记录可能影响改动的边界

### 第三轮：验证

- 搜索是否存在反例或类似实现
- 检查测试、文档和配置
- 确认计划不会遗漏关键路径

## 适用场景

- 大型重构前
- 修复跨模块 bug
- 接手陌生代码库
- 让 subagent 只返回高价值事实

## 输出格式

```markdown
## 已确认事实
- [事实 + 来源文件]

## 相关文件
- [路径：原因]

## 不确定点
- [需要继续确认的问题]

## 下一步检索
- [下一轮要查什么]
```

## 反模式

- 一次读取大量无关文件
- 只搜索一个关键词就下结论
- 忽略测试和配置
- 没有记录证据来源
