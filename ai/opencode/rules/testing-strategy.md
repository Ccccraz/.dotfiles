# 测试策略

## 目标

建立可持续的测试体系，确保变更可验证、可回归、可发布。

## 必须：验证变更

- [ ] 优先运行与改动最相关的测试或检查
- [ ] 新功能和缺陷修复应补充能覆盖核心行为的测试
- [ ] 若无法运行测试，必须说明原因和剩余风险

## 建议：测试分层

- [ ] 单元测试覆盖纯逻辑、工具函数、组件
- [ ] 集成测试覆盖 API、数据库、外部依赖边界
- [ ] E2E 测试覆盖关键用户路径

## 流程：测试失败排查

1. 先确认失败是否能稳定复现
2. 检查测试隔离性和 mock 是否正确
3. 优先修实现而非修测试（除非测试本身有误）
4. 修复后重新运行同一检查确认通过
