# 交付流程

## 目标

规范从开发到提交再到 PR 的交付路径，确保流程一致、信息完整。

## 必须：使用 `commit` 技能执行提交

- [ ] 提交时必须使用 `commit` 技能

## 流程：合并请求

1. 分析完整提交历史（不只看最新提交）
2. 使用 `git diff [base-branch]...HEAD` 检查全量变更
3. 起草完整 PR 摘要
4. 添加测试计划与待办事项
5. 新分支首次推送使用 `-u`

## 流程：功能实现

1. 先规划: 使用 `planner` 识别依赖和风险并分阶段拆解
2. 小步实现: 优先做最小正确变更，避免无需求的兼容层和重构
3. 代码审查: 使用 `code-reviewer`，先修严重和高优先级问题
4. 验证结果: 运行相关测试、类型检查或构建命令
5. 提交推送: 使用 `commit` 技能完成提交并同步测试结果
