# 安全治理

## 目标

定义提交前和发布前的统一安全基线，降低密钥泄露、注入、越权、数据暴露等风险。

## 必须：提交前强制安全检查

- [ ] 无硬编码密钥（API key、密码、token）
- [ ] 所有用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（HTML 已清洗）
- [ ] CSRF 防护已启用
- [ ] 认证/授权已验证
- [ ] 错误信息不泄露敏感数据

## 必须：密钥管理

- [ ] 密钥仅通过环境变量注入
- [ ] 缺失关键密钥时必须快速失败
- [ ] 禁止将 `.env` 类敏感文件纳入版本管理

## 流程：安全事件响应

1. 立即停止当前变更合并
2. 触发 `security-reviewer` 进行审查
3. 先修复 CRITICAL 问题再恢复开发
4. 轮换已暴露的密钥或凭证
5. 在全仓库执行同类问题扫描

## 检查清单

- [ ] 密钥管理合规
- [ ] 输入验证完整
- [ ] 认证与授权路径可验证
- [ ] 用户可见错误信息无敏感字段
