# 代码质量

## 目标

通过统一编码规范保障代码可读性、可维护性和可演进性。

## 必须：不可变性

- [ ] 始终创建新对象/新数组，禁止原地修改
- [ ] 需要更新状态时优先使用展开语法或纯函数

## 建议：文件组织

- [ ] 高内聚、低耦合
- [ ] 优先按功能或领域组织
- [ ] 单文件建议 200-400 行，尽量不超过 800 行

## 必须：错误处理

- [ ] 对外部依赖调用使用 `try/catch`
- [ ] 对用户返回可理解错误，对日志保留排障信息

## 必须：输入验证

- [ ] 所有外部输入必须校验
- [ ] 优先使用 schema 驱动校验（如 zod）
- [ ] 校验失败返回稳定、可预测错误结构

## 必须：完成前质量清单

- [ ] 命名清晰、语义明确
- [ ] 函数尽量小（<50 行）
- [ ] 无深层嵌套（>4 层）
- [ ] 无 `console.log`
- [ ] 无硬编码值
- [ ] 无不必要的原地修改
